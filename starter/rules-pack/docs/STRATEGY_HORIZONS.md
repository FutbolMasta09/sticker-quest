# Strategy Horizons

This document keeps short-, medium-, and long-term strategy aligned so daily work stays focused.

## Short Term (Next 30 Days)

Primary objective: finish current phase milestones with minimal rework.

- Clear top 3 phase blockers weekly.
- Keep one active build focus at a time.
- Run device testing cadence on milestone-worthy changes.
- Keep roadmap and forecast docs current at milestone close.

Success signal:
- Phase checklist completion trend is steady week over week.

## Medium Term (3-6 Months)

Primary objective: launch readiness and stable quality loop.

- Complete Phase 1, Phase 2, and Phase 3 with test evidence.
- Reduce launch debt: fix recurring bugs, stabilize onboarding, clean lint debt.
- Maintain content pipeline so code never waits on content.
- Protect schedule using safe-vs-actual forecasting updates.

Success signal:
- Libby private launch is on track with no critical blockers.

## Long Term (6-18 Months)

Primary objective: scale from one project to a repeatable builder system.

- Launch Kindergarten experience and stabilize support loop.
- Use rules-pack system to bootstrap future projects quickly.
- Expand grade roadmap only after launch stability is confirmed.
- Sequence portfolio work: one active build + one idea lane.

Success signal:
- New projects start with reusable standards in under 30 minutes.

## Do Not Do Yet (Scope Protection)

- Do not open multiple active code projects before Sticker Quest launch is stable.
- Do not add heavy process tools unless current lightweight docs fail.
- Do not expand major content scope without updating timeline and forecast.
