Idea debt cap
Maximum 3 ideas in active status at any moment — where "active" means you have written code or shipped any output in the last 14 days. If a 4th idea becomes active, one of the existing three must move to Backlog or be archived before the new one starts. No exceptions for "I'll just prototype it quickly."
Current active slot recommendation: Prompt Forge only. Time Loop Inbox enters active only after Prompt Forge is in daily personal use.
Monthly promote / archive rule
On the last Friday of each month, spend 20 minutes reviewing every idea tagged Build Now or Build Later against one question: has my reason for not starting this changed? If yes, promote. If an idea has been in Build Now for 60+ days with no first task started, it auto-moves to Backlog — the intent was there but the will wasn't. A Backlog entry is not a failure; it is honest inventory.
An idea in Build Now that never gets a first 2-hour task in 60 days is a wish, not a plan. Move it to Backlog on day 61 — no negotiation.
Validation-first rule
Every idea must pass a core mechanic test before any infrastructure is built. Specifically:
Prompt Forge
Score 5 real prompts with a hand-written rubric before writing a single line of code. If scoring feels useful, build. If it feels like a tax, the product has no argument.
Time Loop Inbox
Design the full 3-loop clue graph on paper and have one person attempt to solve it from the written description. If they can, build. If they can't, redesign before coding.
Inventory Oracle
Hand-write 3 inventory puzzles and have one person try to solve them verbally. If the deduction mechanic produces a satisfying reveal in all 3, build.
Mythwright
Build and use Lore Atlas (H1) in a real worldbuilding session with 20+ entries before touching D3 or Supabase. The flat list version must earn the graph.
Voice DM
Complete 3 text-only sessions with localStorage persistence across different days before adding ElevenLabs. Voice without persistence is a demo, not a product.
Exit criteria — universal reminder
These apply to every active idea in the portfolio, regardless of score or prior investment:
  • If the core mechanic is not fun or useful with static/hardcoded content, adding AI will not fix it. Stop at the hardcoded version and redesign or archive.
  • If you have not shown the v1 to one person outside your own head within 2 weeks of first code, the project is in a bubble. Show it or question whether you believe in it.
  • If any idea requires a system (auth, graph, real-time DB) before a single core loop is playable or usable, you are building infrastructure before product. Stop. Reduce scope until the core loop works with zero infrastructure.
  • If the AI output for an idea is inconsistent in a way that breaks the core promise on more than 30% of runs, that idea is not ready for users. Fix the prompt engineering or move to Backlog.
  • If after 3 weeks you are not using the tool yourself (for tools) or replaying the game yourself (for games), the product has no internal advocate. Archive it honestly.
The one exit criteria that never gets enforced but should: if the idea has been in your head longer than it has been in any user's hands, the psychological investment in the idea exceeds its actual value. This is the most common solo builder trap in this catalog. The cure is shipping the worst possible version as fast as possible — not planning a better one.
