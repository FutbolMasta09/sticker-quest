# Postmortem Template

Use this after any major bug, missed deadline, or surprise that cost meaningful time. Takes 5 minutes. Feeds directly back into the rules and lessons so it never happens the same way twice.

## How to Use

Copy the template below, fill it in, and save it as a new dated section at the bottom of this file. Then check whether a new rule or lesson is warranted.

---

## Template (copy this section)

```
## [Short title for what went wrong]
**Date:** YYYY-MM-DD
**Project:** [Project name]

**What happened:**
[One or two sentences. Plain English.]

**What we expected:**
[What should have happened instead.]

**Root cause:**
[The real reason it went wrong. Be honest. "I didn't check X" is a valid answer.]

**What we changed:**
[What was fixed, updated, or added as a result.]

**New rule (if any):**
[If this warrants a new Cursor rule or lesson, describe it here. Add it to LESSONS_LEARNED.md or create a new .mdc file.]
```
