# API Reset Transition Checklist

Use this on reset week (target window: 2026-04-16).

## 1) Confirm capacity reset

- Verify API usage has reset in Cursor billing.
- Confirm premium models are selectable again.

## 2) Restore model routing

- Re-enable normal model routing for complex coding tasks.
- Keep Auto + Composer as fallback, not default.

## 3) Backlog triage after cap

- List deferred deep-reasoning tasks from PATCH_NOTES.
- Rank top 3 by launch impact.
- Execute only one high-risk deferred task in the first post-reset session.

## 4) Keep what worked

- Keep Monday check-in gate if it improved quality.
- Keep one-objective-per-session rule if rework declined.

## 5) Retrospective log

Record in PATCH_NOTES:
- What the cap period protected well
- What slowed delivery
- Which guardrails stay permanent
