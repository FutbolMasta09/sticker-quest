# Planning Cadence

Simple recurring rhythm to stay on track without process overload.

## Weekly Review (20 Minutes)

Run once per week.

1. Check roadmap: what moved, what stalled.
2. Pick top 1-3 priorities for next week.
3. Note one risk to reduce this week.

Output:
- Updated priorities in chat
- One risk mitigation action

## Milestone Review (30 Minutes)

Run when a phase milestone is completed.

1. Update `ROADMAP_TIMELINE.md`.
2. Update `ESTIMATION_RETROSPECTIVE.md` (safe vs actual delta).
3. Update `FORECAST_BASELINES.md` only if patterns repeat.
4. Run rule upgrade pass and rebuild rules pack.

Output:
- Forecast accuracy update
- Updated multipliers (if justified)

## Monthly Strategy Refresh (30 Minutes)

Run once per month.

1. Review `STRATEGY_HORIZONS.md` short/medium/long sections.
2. Confirm current horizon priorities are still realistic.
3. Move one deferred item forward only if launch risk stays controlled.

Output:
- Refreshed horizon priorities
- Clear "do now vs later" boundary
